# Content Data Structure

本ドキュメントは、「いそしぎ忘備録」で扱うコンテンツ（記事・スライド・ストーリー・イベント参加記録）のデータ構造を定義する。

## 共通スキーマ（Common Schema）

すべてのコンテンツタイプで共通して使用するメタデータフィールド。

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | Yes | コンテンツのタイトル。SEO・OGP・一覧表示に使用。 |
| `description` | string | Yes | コンテンツの概要。SEO・OGP・一覧表示に使用。最大200文字程度を推奨。 |
| `publishedAt` | ISO 8601 Date | Yes | 公開日。ソートやフィード生成に使用。 |
| `updatedAt` | ISO 8601 Date | No | 更新日。未指定の場合は `publishedAt` と同値とみなす。 |
| `tags` | string[] | No | コンテンツを分類するタグ。一覧フィルタや関連コンテンツ表示に使用。 |
| `draft` | boolean | No | `true` の場合、ビルド時に公開対象から除外する。デフォルトは `false`。 |
| `coverImage` | string | No | OGP画像やカード表示用の画像パス。未指定の場合はデフォルト画像を使用。 |

> **メタデータの表示方針**: すべてのメタデータフィールドは「一覧表示用」「詳細表示用」に分けず、共通の概要パネルに表示する。詳細ページの上部にも概要パネルを配置し、メタデータを重複して表示する。

## 記事（Article）

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| `series` | string | No | 連載シリーズ名。同じシリーズ名を持つ記事をグループ化して表示する際に使用。 |
| `seriesOrder` | number | No | シリーズ内の順序。`series` 指定時に使用。 |

- **配置ディレクトリ**: `src/content/articles/`
- **ファイル形式**: `.mdx`
- **ファイル命名**: `slug.mdx`（URLパスに使用されるスラッグ）

## イベント参加記録（EventRecord）

イベントの参加履歴等の簡単な説明を持つコンテンツ。

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| `eventName` | string | Yes | イベント名。 |
| `eventDate` | ISO 8601 Date | Yes | イベントの開催日。 |
| `location` | string | No | 開催場所または開催形式。`online` or `tokyo` |

- **配置ディレクトリ**: `src/content/events/`
- **ファイル形式**: `.mdx`
- **ファイル命名**: `slug.mdx`（URLパスに使用されるスラッグ）
- **ページ生成の有無**: 個別HTMLページは生成しない。共通一覧（`/pages`）に概要パネルとして表示するのみとする。

## スライド（Slide）

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| `theme` | string | No | スライドのテーマ識別子。未指定の場合はデフォルトテーマを使用。 |

- **配置ディレクトリ**: `src/content/slides/`
- **ファイル形式**: `.mdx`
- **ファイル命名**: `slug.mdx`（URLパスに使用されるスラッグ）

## ストーリー（Story）

一連の流れのあるコンテンツ群と、その背景情報を持つコンテンツ。詳細ページでは、`storyFlow` に含まれる各項目を概要パネルとして縦に並べ、その下に本文（MDX）を配置する。

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| `storyFlow` | string[] | Yes | 流れのあるコンテンツ群の概要リスト。各要素は見出し・短い説明文・関連リンクなどを含む文字列。詳細ページでは各項目をカード形式で縦に一覧表示し、その下に本文を配置する。 |

- **配置ディレクトリ**: `src/content/stories/`
- **ファイル形式**: `.mdx`
- **ファイル命名**: `slug.mdx`（URLパスに使用されるスラッグ）

## バリデーションルール

1. **必須フィールド欠如**: 必須フィールドが欠落している場合、ビルド時に警告またはエラーを出力する。
2. **スラッグ重複**: `articles/`, `slides/`, `stories/`, `events/` 内でファイル名（スラッグ）が重複する場合、ビルド時にエラーを出力する。
3. **無効な日付形式**: `publishedAt`, `updatedAt`, `eventDate` が ISO 8601 Date でない場合、ビルド時にエラーを出力する。
4. **不正なタグ**: `tags` に空文字列や重複値が含まれる場合、ビルド時に警告を出力し、空文字列を除外・重複を除去して処理する。
