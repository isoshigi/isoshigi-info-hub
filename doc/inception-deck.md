# 我々はなぜここにいるのか

完璧を求めすぎて公開をためらう傾向があり、検証結果、試行記録、未公開コード、活動記録などが溜まっている。そこで、それらを活用し、「いそしぎ忘備録」というアウトプット共有サイトを作り、迅速かつ簡単に適切なフォーマットでコンテンツにして公開する。

本プロジェクトは、「成長を加速する行動マニフェスト2026」の考え方を実践するための土台である。

# エレベーターピッチ

技術的なアウトプットをしたい個人開発者・技術者向けの、  
「いそしぎ忘備録」は、  
個人の学びや検証結果・活動記録を公開するアウトプット共有サイトです。  
これは完成を待たずに気軽に出せる場所であり、過去の記録を次の挑戦に活かせるため、  
Zenn や Qiita とは異なり、  
自分専用のプラットフォームとしてスライドや記事など多様なフォーマットで、自分のペースで蓄積していくことが備わっています。

# やらないことリストをつくる

- CMS 的な編集・管理機能はやらない（書いてビルドして公開するだけ）
- 他者向けのコメント・SNS 連携機能はやらない（ただし、X への投稿ボタンは除く）
- 重いアニメーション・複雑なインタラクティブ機能は入れない
- 完璧なデザインシステムはやらない
- 汎用的なツール化はやらない
- SEO 最適化やアクセス解析を完璧に仕上げることはやらない。ただし、価値提供のため基本的な SEO 対応は行う。完璧を狙いすぎて公開を止めることだけはやらない。

# 解決案を描く

- Astro + MDXで記事とスライドを同じフォーマットで執筆する
- Cloudflare R2 + CDNにデプロイし、高速・安価・シンプルに公開する
- Tailwind CSS でシンプルな装飾を施す
- Cloudflare Web Analytics で軽量なアクセス解析を行う
- 学びの記録 → 整理・執筆 → コンテンツ公開 → フィードバック → 次の挑戦というサイクルを回す土台を構築する
- 一定のリズムで形にするための「書いてデプロイする」最小手順を確立する
- Playwright を用いてビルド時に OGP 画像を自動生成したり、スライドを画像化して PDF に結合して出力する

# 夜も眠れなくなるような問題は何だろう？

- R2 + CDN へのデプロイ自動化が面倒で、公開のハードルが下がらない
  - 一時的な手動デプロイを許容する
- 他の責務との両立が難しく、リズムが途切れて公開をためらう傾向が再発する
  - 軽量なコンテンツ形式を許容する

# 何をあきらめるのかをはっきりさせる

- **スコープ**: 自分に必要な機能のみに絞る。汎用的なツール化は求めない。
- **予算**: 極力抑える。Cloudflare の無料・低コストプランを活用する。
- **納期**: 完璧を待たずに小さく早く公開することを最重視する。
- **品質**: 基本的な品質は担保するが、完璧を狙いすぎて公開を止めることはしない。
