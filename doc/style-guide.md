# Style Guide

本ドキュメントは、スタイルの判断基準を定める。細部の仕様を網羅するものではない。

## 1. 基本方針

- **実用主義**: 読みやすさと公開の速さを最優先する。完璧を待たない。
- **最小限のコンポーネント**: ボタン・パネル・ヘッダー・フッター・サイドバーで原則すべてを構成する。新しいコンポーネントを作る前に、既存の組み合わせで済まないか考える。
- **無駄なし**: アニメーションや複雑なインタラクションは原則入れない。

## 2. 色の判断基準

Tailwind CSS の `sky` と `slate` のみを使用する。独自の色定義は行わない。

- **メインの強調**（リンク、ボタン、現在位置）: `sky` 系
- **背景・テキスト・枠線**（情報の下地と階層）: `slate` 系
- **アクセント**（タグ、ハイライト）: `sky` の薄い色を `slate` の濃いテキストと組み合わせる

色を選ぶ際は、上記2色軸に収まるかを最初に確認する。

## 3. タイポグラフィの判断基準

- **フォント**: システムフォントスタックのみ。Webフォントの読み込みは行わない。
- **階層**: 情報の優先度を、サイズと字重で表現する。極端なサイズの乱用は避ける。
- **行間**: 本文は読みやすい程度に広めに取る。

## 4. レイアウトの判断基準

- **全ページ共通レイアウト**: ヘッダー・フッター・サイドバーは全ページで共通とする。
- **最大幅**: ありすぎずなさすぎず。極端に広げない。
- **余白**: 要素間の距離で情報のグルーピングを示す。詰めすぎず、離しすぎず。
- **モバイルファースト**: まずモバイルを想定し、必要に応じて広い画面用の調整を加える。

## 6. やらないこと

以下は v0.1 では行わない。

- ダークモード対応（v0.2 以降で検討）
- アニメーションやトランジション（ただし、hover での色変化など最低限のフィードバックは許容する）
- 完璧なデザインシステムやコンポーネントライブラリ化
- 重いインタラクティブ機能

## 7. 迷ったときの基準

スタイルの判断に迷った場合、以下を優先順に確認する。

1. **本書の「基本方針」に反していないか**
2. **既存のコンポーネントで実現できないか**
3. **sky と slate の範囲で収まるか**
4. **モバイルで見たときに支障がないか**

上記を満たすなら、細部の揺らぎは許容して先に進む。完璧を待たない。
