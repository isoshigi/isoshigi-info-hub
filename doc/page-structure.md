# Page Structure

本ドキュメントは、いそしぎ忘備録の画面構成とURLルーティングを定義する。

## 全ページ共通レイアウト

すべてのページで以下のレイアウトを共有する。

- **ヘッダー**: サイトタイトル、グローバルナビゲーション
- **フッター**: 著作権表示、SNSリンクなど
- **サイドバー**: （v0.1 では最小限の内容。将来の拡張を見据えてレイアウトのみ確保）

## URL ルーティングとページ役割

| URL | ページ名 | 役割 | 備考 |
|---|---|---|---|
| `/` | トップページ | 最新のコンテンツ数件、プロフィール、サイト概要への導線を表示する。 | v0.1 |
| `/pages` | 全コンテンツ共通一覧 | 技術記事・イベント参加記録・スライド・ストーリーを時系列で混在して一覧表示する。各アイテムは概要パネルで表示する。 | v0.1 |
| `/articles/[slug]` | 技術記事詳細 | 概要パネル + MDX本文を表示する。 | v0.1 |
| `/slides/[slug]` | スライド詳細 | 概要パネル + スライドビューアを表示する。 | v0.2 |
| `/stories/[slug]` | ストーリー詳細 | 概要パネル + ストーリー本文を表示する。 | v0.2 |
| `/events/[slug]` | **（なし）** | 個別HTMLページは生成しない。イベント参加記録は `/pages` 共通一覧に概要パネルとして表示するのみ。 | — |

## ページ構成のルール

1. **一覧と詳細の分離**: 共通一覧（`/pages`）以外に、コンテンツタイプ別の個別一覧ページは設けない。
2. **概要パネルの配置**:
   - 共通一覧（`/pages`）: 各アイテムを概要パネル（カード形式）で表示
   - 詳細ページ（`/articles/[slug]`、`/slides/[slug]`、`/stories/[slug]`）: ページトップに概要パネルを配置し、その下にコンテンツ本体を表示
3. **イベント参加記録の扱い**: コンテンツファイルとしては存在するが、Astro の `getStaticPaths` などで個別ページの生成対象外とする。スラッグの重複チェックは他コンテンツと同様に行う。
4. **SEO メタ情報**: すべてのページで `<title>`、`<meta name="description">`、OGP 画像を設定する。詳細ページはコンテンツ個別のメタ情報を使用する。
