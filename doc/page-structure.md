# Page Structure

本ドキュメントは、いそしぎ忘備録の画面構成とURLルーティングを定義する。

## 全ページ共通レイアウト

すべてのページで以下のレイアウトを共有する。

- **ヘッダー**: サイトタイトル、グローバルナビゲーション
- **フッター**: 著作権表示、SNSリンクなど
- **サイドバー**: （v0.1 では最小限の内容。将来の拡張を見据えてレイアウトのみ確保）

## URL ルーティングとページ役割

| URL | ページ名 | 役割 | 備考 |
|---|---|---|---|
| `/` | トップページ | 最新のコンテンツ数件、プロフィール、サイト概要への導線を表示する。 | v0.1 |
| `/pages` | 全コンテンツ共通一覧 | 技術記事・スライド・ストーリーを時系列で混在して一覧表示する。各アイテムは概要パネルで表示する。 | v0.1 |
| `/articles/[slug]` | 技術記事詳細 | 概要パネル + MDX本文を表示する。 | v0.1 |
| `/logs` | 軽量コンテンツ一覧 | イベント参加記録など、フロントマターのみの軽量コンテンツを一覧表示する。 | v0.2 |
| `/slides/[slug]` | スライド詳細 | 概要パネル + スライドビューアを表示する。 | v0.3 |
| `/stories/[slug]` | ストーリー詳細 | 概要パネル + ストーリー本文を表示する。 | v0.3 |
| `/events/[slug]` | **（なし）** | 個別HTMLページは生成しない。イベント参加記録は `/events` 一覧に表示するのみ。 | — |

## ページ構成のルール

1. **一覧と詳細の分離**: コンテンツは「通常コンテンツ（Standard Content）」と「軽量コンテンツ（Lightweight Content）」の2つに分類する。
   - 通常コンテンツ（共通スキーマ準拠、MDX本文あり）は `/pages` に集約する。
   - 軽量コンテンツ（独立スキーマ、フロントマターのみ）は `/logs` に集約する。
   - この分離は `tmp/manifest2026.md` の理念に基づく：中間成果物はそのまま公開せず、抽出した知見を「物語として完成させた」通常コンテンツと、生の記録として残す軽量コンテンツを区別する。
2. **概要パネルの配置**:
   - 共通一覧（`/pages`）: 各アイテムを概要パネル（カード形式）で表示
   - 詳細ページ（`/articles/[slug]`、`/slides/[slug]`、`/stories/[slug]`）: ページトップに概要パネルを配置し、その下にコンテンツ本体を表示
3. **軽量コンテンツの扱い**: 共通スキーマとは独立した構造で、`/logs` に一覧表示する。個別ページは生成せず、スラッグも URL パスとして使用しない。イベント参加記録（`events`）は軽量コンテンツの一例であり、将来的に他の種類が追加される可能性がある。
4. **SEO メタ情報**: すべてのページで `<title>`、`<meta name="description">`、OGP 画像を設定する。詳細ページはコンテンツ個別のメタ情報を使用する。**OGP 画像はビルド時に自動生成される（`/img/.../og.png`）。**
